# content/

Put your **fiesta / Mimir content project** here — the git-tracked JSON tables
plus `mimir.json` (the project marker the toolkit looks for). This is what the
pipeline runs `fiesta build` against to produce the server SHN/txt.

A typical project looks like:

```
content/
├── mimir.json                 # project marker + environment config
├── data/                      # the JSON tables (one file per table) — commit these
├── environments/<env>.json    # per-environment import/build config
└── build/                     # generated by `fiesta build` (gitignored)
```

Create one with the toolkit (from ik-fiesta-collab):

```bash
fiesta init content
fiesta import      # import your ServerSource/ClientSource into JSON
```

**No copyrighted game data is committed here by this repo** — the JSON tables you
add are yours. The build output (`content/build/`) is gitignored; it's
regenerated by the pipeline on every run.
